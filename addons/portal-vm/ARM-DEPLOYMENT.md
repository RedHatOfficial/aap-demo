# Portal VM on macOS ARM (Apple Silicon)

Complete guide for deploying AAP Self-Service Portal on ARM Macs using x86 emulation.

## Overview

- **Platform:** macOS ARM (Apple Silicon M1/M2/M3)
- **Method:** QEMU x86-64 emulation
- **Performance:** 5-10x slower than native (dev/testing only)
- **Boot time:** 3-10 minutes
- **Access:** Portal UI (primary), tmux console (for debugging)

## Prerequisites

```bash
brew install qemu cdrtools tmux
```

Download portal qcow2:

- URL: https://access.redhat.com/downloads/content/480/ver=2.7/rhel---9/2.7/x86_64/product-software
- File: `ansible-automation-portal-2.2.1-x86_64.qcow2`
- Save to: `~/Downloads/`

## Quick Start

```bash
# Deploy portal VM
./addons/portal-vm/deploy.sh

# Access portal UI (after 3-10min boot)
open https://localhost:8443

# Console access for debugging
tmux attach -t portal-vm
# Login: admin / admin
# Detach: Ctrl-b d

# Stop VM
./addons/portal-vm/deploy.sh --delete
```

## Access Methods

### Portal UI (Primary)

```bash
# Access after boot completes
open https://localhost:8443

# Or direct URL
https://localhost:8443
```

### Console Access (Debugging)

```bash
# Attach to VM console
tmux attach -t portal-vm

# Login credentials
Username: admin
Password: admin

# Detach without stopping VM
Ctrl-b d

# Check services
sudo systemctl status portal postgres devtools

# View logs
journalctl -f -u portal
```

### SSH Access

**Status:** Currently unavailable from macOS host

**Issue:** QEMU user networking not forwarding SSH traffic despite:

- Port 2222 bound and reachable
- VM sshd running and functional
- SSH works inside VM (localhost → localhost)

**Workaround:** Use tmux console access above

## Performance

### Boot Time

- **First boot:** 5-10 minutes
- **Subsequent boots:** 3-5 minutes
- **Bottleneck:** x86 emulation on ARM

### Runtime Performance

- **UI latency:** Noticeable but usable
- **Container startup:** Slow (podman on emulated x86)
- **Recommendation:** Dev/testing only, not production

### CPU Model

- **Uses:** Nehalem (x86-64-v2)
- **Why:** Portal requires SSE4.2, POPCNT, CMPXCHG16B
- **Not using:** HVF acceleration (unavailable for x86 on ARM)

## Configuration

### Cloud-Init

Portal configured via cloud-init on first boot:

```yaml
#cloud-config
ssh_pwauth: true
users:
  - name: admin
    groups: sudo
    lock_passwd: false
    plain_text_passwd: admin
    ssh_authorized_keys:
      - <generated-key>

aap:
  host_url: "https://aap-aap-operator.apps.10.0.2.2.nip.io"
  token: "<aap-admin-password>"
  check_ssl: false
  oauth:
    client_id: "<auto-generated>"
    client_secret: "<auto-generated>"

database:
  type: builtin
```

### OAuth Application

Created automatically in AAP:

- **Name:** portal-vm
- **Type:** confidential
- **Grant:** authorization-code
- **Redirect URI:** https://localhost:8443/api/auth/callback
- **Organization:** 1 (Default)

### Network

QEMU user networking:

- **VM IP:** 10.0.2.15 (internal)
- **Host gateway:** 10.0.2.2 (from VM perspective)
- **AAP URL (from VM):** https://aap-aap-operator.apps.10.0.2.2.nip.io
- **Port forwards:** 443→8443, 80→8080, 22→2222 (SSH broken)

## Troubleshooting

### VM won't boot

```bash
# Check boot progress
tail -f ~/.aap-demo/portal-vm/qemu.log

# Common issues:
# - Corrupted qcow2: re-download from Customer Portal
# - Insufficient RAM: need 8GB free
```

### Portal UI not loading

```bash
# Check if VM booted
tail -50 ~/.aap-demo/portal-vm/qemu.log | grep "login:"

# Check services (from console)
tmux attach -t portal-vm
sudo systemctl status portal postgres devtools
journalctl -u portal --no-pager | tail -50
```

### OAuth login fails

```bash
# From console, check portal config
tmux attach -t portal-vm
sudo cat /etc/portal/configs/app-config/app-config.production.yaml | grep -A5 oauth

# Verify AAP reachable from VM
curl -k https://aap-aap-operator.apps.10.0.2.2.nip.io/api/v2/ping/

# Check OAuth app exists in AAP (from host)
kubectl get -n aap-operator route aap -o jsonpath='{.spec.host}'
curl -sk -u admin:<password> https://<aap-route>/api/gateway/v1/applications/ | jq '.results[] | select(.name=="portal-vm")'
```

### Portal services won't start

```bash
# From console
tmux attach -t portal-vm

# Check podman containers
podman ps -a

# Check container logs
sudo journalctl -u portal -n 100
sudo journalctl -u postgres -n 100

# Restart services
sudo systemctl restart postgres
sudo systemctl restart portal
```

### Console login not working

**Credentials:**

- Username: `admin`
- Password: `admin`

If password doesn't work:

1. Cloud-init may have failed to set password
2. Admin user was already locked
3. Redeploy with fresh qcow2

### Tmux session not found

```bash
# List tmux sessions
tmux list-sessions

# If no portal-vm session:
# VM exited or was never started in tmux
# Check qemu.log for exit reason
tail -50 ~/.aap-demo/portal-vm/qemu.log
```

## Limitations

### SSH Access

- **Cannot SSH from macOS host** (QEMU user networking issue)
- **Console access only** (tmux)
- **Affects:** Direct file transfer, remote debugging
- **Workaround:** Use console + copy/paste

### Performance

- **5-10x slower than native x86**
- **Container operations are slow** (podman pull, start)
- **UI latency noticeable**
- **Not suitable for load testing**

### Resource Usage

- **RAM:** 8GB dedicated to VM
- **CPU:** 4 cores (emulated)
- **Disk:** ~30GB for qcow2 + containers

## Known Issues

1. **SSH port forwarding broken**
   - QEMU user networking not passing SSH traffic
   - Console access works as workaround

2. **Slow boot on ARM**
   - x86-64 emulation inherently slow
   - No fix - architectural limitation

3. **Cloud-init schema warnings**
   - Portal extends cloud-init with custom `aap:` block
   - Warning is non-fatal, config applies correctly
